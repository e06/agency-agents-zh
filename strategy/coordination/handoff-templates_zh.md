# 📋 NEXUS 移交模板

> NEXUS 流水线中各种类型智能体间移交的标准化模板。一致的移交可防止上下文丢失——这是多智能体协调失败的首要原因。

---

## 1. 标准移交模板

用于任何智能体之间的工作转移。

```markdown
# NEXUS 移交文档

## 元数据
| 字段 | 值 |
|-------|-------|
| **来自** | [智能体名称] ([部门]) |
| **转至** | [智能体名称] ([部门]) |
| **阶段** | 阶段 [N] — [阶段名称] |
| **任务参考** | [来自 Sprint Prioritizer 待办事项的任务 ID] |
| **优先级** | [紧急 / 高 / 中 / 低] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 上下文
**项目**: [项目名称]
**当前状态**: [截至目前已完成的工作 — 需具体说明]
**相关文件**:
- [文件/路径/1] — [包含内容]
- [文件/路径/2] — [包含内容]
**依赖项**: [此项工作所依赖的已完成工作]
**约束条件**: [技术、时间或资源约束]

## 交付物请求
**所需内容**: [具体、可衡量的交付物描述]
**验收标准**:
- [ ] [标准 1 — 可衡量]
- [ ] [标准 2 — 可衡量]
- [ ] [标准 3 — 可衡量]
**参考资料**: [规范、设计、先前工作的链接]

## 质量要求
**必须通过**: [此交付物的具体质量标准]
**所需证据**: [完成证明的形式]
**移交至**: [谁接收输出以及他们需要的格式]
```

## 2. QA 反馈循环 — 通过

当 Evidence Collector 或其他 QA 智能体批准任务时使用。

```markdown
# NEXUS QA 裁决: 通过 ✅

## 任务
| 字段 | 值 |
|-------|-------|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发智能体** | [智能体名称] |
| **QA 智能体** | [智能体名称] |
| **尝试次数** | 第 [N] 次，共 3 次 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 裁决: 通过

## 证据
**截图**:
- 桌面端 (1920x1080): [文件名/路径]
- 平板端 (768x1024): [文件名/路径]
- 移动端 (375x667): [文件名/路径]

**功能验证**:
- [x] [验收标准 1] — 已验证
- [x] [验收标准 2] — 已验证
- [x] [验收标准 3] — 已验证

**品牌一致性**: 已验证 — 颜色、排版、间距符合设计系统
**无障碍性**: 已验证 — 键盘导航、对比度、语义化 HTML
**性能**: [测量的加载时间] — 在可接受范围内

## 备注
[任何观察、对未来改进的次要建议或正面反馈]

## 下一步行动
→ Agents Orchestrator: 标记任务完成，推进待办事项中的下一个任务
```

## 3. QA 反馈循环 — 失败

当 Evidence Collector 或其他 QA 智能体拒绝任务时使用。

```markdown
# NEXUS QA 裁决: 失败 ❌

## 任务
| 字段 | 值 |
|-------|-------|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发智能体** | [智能体名称] |
| **QA 智能体** | [智能体名称] |
| **尝试次数** | 第 [N] 次，共 3 次 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 裁决: 失败

## 发现的问题

### 问题 1: [类别] — [严重程度: 紧急/高/中/低]
**描述**: [问题的精确描述]
**预期结果**: [根据验收标准应发生的情况]
**实际结果**: [实际发生的情况]
**证据**: [截图文件名或测试输出]
**修复指引**: [解决问题的具体、可操作指引]
**需修改的文件**: [精确的文件路径]

### 问题 2: [类别] — [严重程度]
**描述**: [...]
**预期结果**: [...]
**实际结果**: [...]
**证据**: [...]
**修复指引**: [...]
**需修改的文件**: [...]

[继续列出所有发现的问题]

## 验收标准状态
- [x] [标准 1] — 通过
- [ ] [标准 2] — 失败 (见问题 1)
- [ ] [标准 3] — 失败 (见问题 2)

## 重试指引
**致开发智能体**:
1. 仅修复上述列出的问题
2. 不要引入新功能或更改
3. 当所有问题解决后重新提交 QA
4. 这是第 [N] 次尝试，最多 3 次

**如果第 3 次尝试失败**: 任务将升级至 Agents Orchestrator
```

## 4. 升级报告

当任务超过 3 次重试尝试时使用。

```markdown
# NEXUS 升级报告 🚨

## 任务
| 字段 | 值 |
|-------|-------|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发智能体** | [智能体名称] |
| **QA 智能体** | [智能体名称] |
| **已用尝试次数** | 3/3 |
| **升级至** | [Agents Orchestrator / Studio Producer] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 失败历史

### 第 1 次尝试
- **发现的问题**: [摘要]
- **应用的修复**: [开发者做了什么更改]
- **结果**: 失败 — [为何仍然失败]

### 第 2 次尝试
- **发现的问题**: [摘要]
- **应用的修复**: [开发者做了什么更改]
- **结果**: 失败 — [为何仍然失败]

### 第 3 次尝试
- **发现的问题**: [摘要]
- **应用的修复**: [开发者做了什么更改]
- **结果**: 失败 — [为何仍然失败]

## 根本原因分析
**任务持续失败的原因**: [对潜在问题的分析]
**系统性问题**: [这是一次性问题还是某种模式？]
**复杂度评估**: [任务范围是否合理？]

## 建议解决方案
- [ ] **重新分配** 给其他开发智能体 ([推荐智能体])
- [ ] **拆分** 为更小的子任务 ([建议的拆分方案])
- [ ] **修改方法** — 需要架构/设计变更
- [ ] **接受** 当前状态并记录限制
- [ ] **推迟** 到未来的冲刺

## 影响评估
**阻塞项**: [哪些其他任务被此任务阻塞]
**时间影响**: [如何影响整体进度]
**质量影响**: [如果接受当前状态存在哪些质量折衷]

## 需做出的决策
**决策者**: [Agents Orchestrator / Studio Producer]
**截止时间**: [需要何时做出决策以避免进一步延误]
```

## 5. 阶段关口移交

在 NEXUS 阶段之间转换时使用。

```markdown
# NEXUS 阶段关口移交

## 过渡
| 字段 | 值 |
|-------|-------|
| **来源阶段** | 阶段 [N] — [名称] |
| **目标阶段** | 阶段 [N+1] — [名称] |
| **关口负责人** | [智能体名称] |
| **关口结果** | [通过 / 失败] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 关口标准结果
| # | 标准 | 阈值 | 结果 | 证据 |
|---|-----------|-----------|--------|----------|
| 1 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |
| 2 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |
| 3 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |

## 延续文档
1. [文档名称] — [下一阶段的用途]
2. [文档名称] — [下一阶段的用途]
3. [文档名称] — [下一阶段的用途]

## 下一阶段的关键约束
- [来自本阶段发现的约束 1]
- [来自本阶段发现的约束 2]

## 下一阶段智能体激活
| 智能体 | 角色 | 优先级 |
|-------|------|----------|
| [智能体 1] | [下一阶段角色] | [立即 / 第 2 天 / 按需] |
| [智能体 2] | [下一阶段角色] | [立即 / 第 2 天 / 按需] |

## 延续风险
| 风险 | 严重程度 | 缓解措施 | 负责人 |
|------|----------|------------|-------|
| [风险] | [P0-P3] | [缓解计划] | [智能体] |
```

```markdown
|---------|-------------|--------|-------------|-------|

| [ID] | [描述] | ✅ 完成 | [N] | [备注] |
```

```markdown
## 描述
**发生情况**: [事故的清晰描述]
**影响范围**: [受影响的人/物及严重程度]
**时间线**:
- [HH:MM] — [事件]
- [HH:MM] — [事件]
- [HH:MM] — [事件]

## 当前状态
**受影响系统**: [列表]
**可用变通方案**: [是/否 — 如是请描述]
**预计解决时间**: [时间估计]

## 已采取措施
1. [采取的行动及结果]
2. [采取的行动及结果]

## 移交上下文
**致下一响应者**:
- [已尝试的方法]
- [尚未尝试的方法]
- [疑似根本原因]
- [需检查的相关日志/指标]

## 利益相关者沟通
**上次更新发送**: [时间戳]
**下次更新截止**: [时间戳]
**沟通渠道**: [更新发布位置]
```

## 使用指南

| 场景 | 使用的模板 |
|-----------|----------------|
| 向其他智能体分配工作 | 标准移交 (#1) |
| QA 批准任务 | QA 通过 (#2) |
| QA 拒绝任务 | QA 失败 (#3) |
| 任务超过 3 次重试 | 升级报告 (#4) |
| 在阶段之间移动 | 阶段关口移交 (#5) |
| 冲刺结束 | 冲刺移交 (#6) |
| 系统事故 | 事故移交 (#7) |
