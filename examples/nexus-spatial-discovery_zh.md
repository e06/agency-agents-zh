## 1. 机会发现

### 如何发现

通过多源网络研究，发现了三个趋同趋势：

- **AI基础设施/编排**是增长最快的软件类别（AI编排市场2026年估值约135亿美元，复合年增长率22%以上）
- **空间计算**（Vision Pro、WebXR）正在成熟，但缺乏杀手级企业应用
- 现有的所有AI工作流工具（LangSmith、n8n、Flowise、CrewAI）都是**平面2D仪表板**

### 概念：Nexus Spatial

一个空间计算中的AI智能体指挥中心——一个VisionOS + WebXR应用程序，为编排、监控和与AI智能体交互提供沉浸式3D指挥中心。用户将智能体流水线可视化为3D节点图，在空间面板中监控实时输出，在3D空间中通过拖放构建工作流，并在共享空间环境中协作。

### 为什么这家代理机构具有独特优势

该代理机构拥有深厚的空间计算专业知识（XR开发者、VisionOS工程师、Metal专家、界面架构师），同时具备完整的工程、设计、营销和运营能力——对于一款既需要空间计算精通又需要企业软件严谨性的产品来说，这是一种罕见的组合。

## 3. 技术架构

**智能体：** 后端架构师

### 系统概览

8服务架构，具有清晰的归属边界，专为水平扩展和提供商无关的AI集成而设计。

```
+------------------------------------------------------------------+
|                     客户端层                                       |
|  VisionOS原生 (Swift/RealityKit)  |  WebXR (React Three Fiber) |
+------------------------------------------------------------------+
                              |
+-----------------------------v------------------------------------+
|                      API网关 (Kong / AWS API GW)                  |
|  限流 | JWT验证 | WebSocket升级 | TLS                           |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                      服务层                                       |
|  认证 | 工作空间 | 工作流 | 编排引擎(Rust) |                       |
|  协作(Yjs CRDT) | 流式传输(WS) | 插件 | 计费                      |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                      数据层                                       |
|  PostgreSQL 16 | Redis 7集群 | S3 | ClickHouse | NATS            |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                    AI提供商层                                     |
|  OpenAI | Anthropic | Google | 本地模型 | 自定义插件               |
+------------------------------------------------------------------+
```

### 技术栈

| 组件 | 技术 | 理由 |
|-----------|------------|-----------|
| 编排引擎 | **Rust** | 亚毫秒调度，零GC暂停，智能体沙箱的内存安全 |
| API服务 | TypeScript / NestJS | CRUD密集型服务的开发效率 |
| VisionOS客户端 | Swift 6, SwiftUI, RealityKit | 一流的空间计算与Liquid Glass |
| WebXR客户端 | TypeScript, React Three Fiber | 生产级WebXR与React组件模型 |
| 消息代理 | NATS JetStream | 轻量级，精确一次投递，比Kafka更简单 |
| 协作 | Yjs (CRDT) + WebRTC | 无冲突的并发3D图形编辑 |
| 主数据库 | PostgreSQL 16 | JSONB用于灵活配置，行级安全用于租户隔离 |

### 核心数据模型

14个表覆盖：
- **身份与访问：** users、workspaces、team_memberships、api_keys
- **工作流：** workflows、workflow_versions、nodes、edges
- **执行：** executions、execution_steps、step_output_chunks
- **协作：** collaboration_sessions、session_participants
- **凭证：** provider_credentials（AES-256-GCM加密）
- **计费：** subscriptions、usage_records
- **审计：** audit_log（仅追加）

### 节点类型注册表

```
Built-in Node Types:
  ai_agent          -- 使用提示词调用AI提供商
  prompt_template   -- 使用变量渲染模板
  conditional       -- 基于表达式路由
  transform         -- 沙箱代码片段(JS/Python)
  input / output    -- 工作流入口/出口点
  human_review      -- 暂停等待人工审批
  loop              -- 重复子图
  parallel_split    -- 分发到多个分支
  parallel_join     -- 等待分支完成
  webhook_trigger   -- 外部HTTP触发器
  delay             -- 定时暂停
```

### WebSocket通道

通过WSS进行实时流式传输：
- 每通道序列号用于排序
- 间隙检测与重放请求
- 落后超过1000个事件时进行快照恢复
- 针对低功耗设备的客户端节流

### 安全架构

| 层级 | 机制 |
|-------|-----------|
| 用户认证 | OAuth 2.0（GitHub、Google、Apple）+ 邮箱/密码 + 可选TOTP MFA |
| API密钥 | SHA-256哈希，范围限定，可选过期 |
| 服务间通信 | 通过服务网格的mTLS |
| WebSocket认证 | 一次性票据，30秒过期 |
| 凭证存储 | 信封加密（AES-256-GCM + AWS KMS） |
| 代码沙箱 | gVisor/Firecracker微虚拟机（无网络，256MB内存，30秒CPU） |
| 租户隔离 | PostgreSQL行级安全 + S3 IAM策略 + NATS主题范围限定 |

## 4. 品牌策略

**智能体：** 品牌守护者

### 定位

**品类创造胜过品类竞争。** Nexus Spatial定义了一个新品类——**空间AI运维（SpatialAIOps）**——而不是在拥挤的AI可观测性仪表板领域争夺位置。

**定位陈述：** 对于管理复杂AI智能体工作流的技术团队，Nexus Spatial是沉浸式3D指挥中心，提供智能体编排的空间感知，与平面2D仪表板不同，因为空间计算将监控从阅读仪表板转变为身临其境地感知基础设施。

### 名称验证

"Nexus Spatial"**验证为强势：**
- "Nexus"连接到NEXUS编排框架（专家网络，策略统一）
- "Nexus"独立意为"中心连接点"——完美适用于指挥中心
- "Spatial"是苹果和行业已标准化的行业描述符
- 语音平衡：三个音节，然后两个
- **需采取行动：** 在尼斯分类第9、42和38类中进行商标清算

### 关键发现：调试是杀手级用例

运行时追踪在工作流结构上的空间叠加解决了一个真正的、量化的痛点，没有2D工具能很好处理。这个工作流应获得最多的设计和工程投入。

### 关键设计洞察

空间为**结构性**任务（放置、连接、重排节点）增加价值，但为**参数性**任务（文本输入、配置）制造摩擦。界面必须无缝融合空间和2D模式——锚定到空间位置的2D面板。

### 7项设计原则

1. **空间必须证明其价值**——如果2D更清晰，就用2D。每次评审都应问："这用平面会更好吗？"
2. **先可扫视再可检查**——关键信息通过颜色、大小、运动、位置在2秒内可感知
3. **免手是基准**——眼动+语音覆盖所有阅读/导航操作；手增加精度但非必需
4. **尊重认知重力**——扩展2D心智模型（从左到右流程），不取代；z轴增加分层
5. **渐进式空间复杂度**——新用户几乎从2D开始；空间能力随信心增长而展现
6. **物理隐喻，数字能力**——节点被"拾起"（物理）但也可复制和版本化（数字）
7. **静默是功能**——健康系统感觉平静；颜色和运动表示偏离正常

## 9. 空间界面架构

**智能体：** XR界面架构师

### 指挥剧场

工作空间组织为围绕用户的曲面剧场：

```
                        概览天篷
                     (流水线拓扑)
                    ~~~~~~~~~~~~~~~~~~~~~~~~
                   /                        \
                  /     聚焦弧 (120度)         \
                 /    主节点图工作区            \
                /________________________________\
               |                                  |
    左侧       |        用户位置                   |       右侧
    工具       |        (原点 0,0,0)              |       工具
    栏         |                                  |       栏
               |__________________________________|
                \                                /
                 \      架子 (视线以下)          /
                  \   智能体状态，快捷工具       /
                   \_________________________ /
```

- **聚焦弧**（120度，1.2-2.0米）：主节点图工作空间
- **概览天篷**（上方，2.5-4.0米）：小型流水线拓扑 + 健康热力图
- **工具栏**（左/右翼）：智能体库、监控、日志
- **架子**（视线以下，0.8-1.0米）：运行/停止、撤销/重做、快捷工具

### 三层深度系统

| 层级 | 深度 | 内容 | 不透明度 |
|-------|-------|---------|---------|
| 前景 | 0.8 - 1.2米 | 活跃面板、检查器、模态框 | 100% |
| 中景 | 1.2 - 2.5米 | 节点图、连接、工作空间 | 100% |
| 背景 | 2.5 - 5.0米 | 概览地图、环境状态 | 40-70% |

### 3D节点图

**数据流向用户流动。** 节点按执行顺序沿z轴排列：

```
用户 (此处)
  z=0.0m   [输出节点]     -- 结果
  z=0.3m   [转换节点]  -- 处理器
  z=0.6m   [智能体节点]      -- LLM调用
  z=0.9m   [检索节点]  -- RAG, API
  z=1.2m   [输入节点]      -- 触发器
```

并行分支水平展开（x轴）。条件分支垂直展开（y轴）。

**节点表示（3级细节）：**
- **LOD-0**（静止，>1.5米）：12x8cm磨砂玻璃矩形，带类型图标、名称、状态发光
- **LOD-1**（悬停，400ms注视）：扩展到14x10cm，显示端口、上次运行信息
- **LOD-2**（选中）：滑至前景，扩展到30x40cm详情面板，支持实时配置编辑

**连接作为发光管道：**
- 静止时直径4mm，传输数据时8mm
- 按数据类型着色（白色=文本，青色=结构化，品红=图像，琥珀=音频，绿色=工具调用）
- 动画粒子显示流向和速度
- 同层间超过3条并行时自动捆绑

### 7种智能体状态

| 状态 | 边缘发光 | 内部 | 声音 | 粒子 |
|-------|-----------|----------|-------|-----------|
| 空闲 | 稳定绿色，低 | 静态磨砂玻璃 | 无 | 无 |
| 排队 | 脉冲琥珀色，1Hz | 微弱旋转 | 无 | 输入端缓慢漂移 |
| 运行中 | 稳定蓝色，中 | 动画闪烁 | 柔和空间嗡鸣 | 连接上快速流动 |
| 流式传输 | 蓝色 + 输出流 | 闪烁 + 文本片段 | 嗡鸣 | 文本片段向前流动 |
| 已完成 | 白色闪烁，然后绿色 | 静态 | 完成提示音 | 无 |
| 错误 | 脉冲红色，2Hz | 红色色调 | 警报音（一次） | 无 |
| 已暂停 | 稳定琥珀色 | 冻结帧 + 暂停图标 | 无 | 原地冻结 |

| 过渡 | 持续时间 | 关键动作 |
|-----------|----------|------------|
| 概览到聚焦 | 600ms | 相机漂移到目标，其他区域淡至30% |
| 聚焦到详情 | 500ms | 节点滑向前方，展开，连接高亮 |
| 详情到概览 | 600ms | 面板折叠，节点后退，完整拓扑可见 |
| 区域切换 | 500ms | 当前滑出侧面，新滑入 |
| 窗口到沉浸式 | 1000ms | 边框溶解，节点扩展到完整空间位置 |

### 舒适度措施

- 无用户操作不进行相机发起的运动
- 稳定地平线（水平面从不倾斜）
- 主要交互在0.8-2.5米内，视线±15度
- 45分钟后休息提示（环境照明变化，非模态）
- 快速运动期间周边暗角
- 所有常用控件可在手臂自然下垂时访问（仅手腕/手指）

## 10. 跨智能体综合

### 所有8个智能体的共识点

1. **2D优先，空间其次。** 每个智能体独立得出此结论。先构建出色的网页仪表板，然后逐步添加空间能力。

2. **调试是杀手级用例。** 产品趋势研究员、用户体验研究员和XR界面架构师都汇聚于此：运行时追踪在工作流结构上的空间叠加是3D真正超越2D的地方。

3. **WebXR优于VisionOS用于初期覆盖。** Vision Pro约100万台的装机量无法支撑业务。浏览器中的WebXR是分发突破口。

4. **"作战室"协作场景。** 多个智能体强调协作式事件响应是最强的空间价值主张——团队进入共享3D空间一起调试故障流水线。

5. **渐进式披露至关重要。** 用户体验研究、空间UI和支持都强调空间复杂度必须逐步展现，绝不一次性堆给新用户。

6. **语音作为高级用户加速器。** 用户体验研究员和XR界面架构师都将语音命令识别为"空间计算的命令行"——对无障碍和专家效率至关重要。

### 需解决的关键张力

| 张力 | 位置A | 位置B | 需解决 |
|---------|-----------|-----------|-------------------|
| **定价** | 增长黑客：29-59美元/用户/月 | 趋势研究员：99-249美元/用户/月 | 测试版中A/B测试 |

### 本演练所展示的

这份发现文档由8个专业智能体并行运行产生，每个都为共同目标带来深厚的领域专业知识。智能体独立得出一致的结论，同时浮现出任何单一通才难以产生的领域特定洞察：

- **产品趋势研究员**发现了令人清醒的Vision Pro销售数据，重新构建了整个策略
- **后端架构师**设计了一个营销导向团队不会考虑的Rust编排引擎
- **品牌守护者**创造了一个新品类（"SpatialAIOps"）而非在现有品类中竞争
- **用户体验研究员**发现空间计算为参数任务制造摩擦——一个反直觉的发现
- **XR界面架构师**设计了"数据向用户流动"的拓扑，映射到自然空间认知
- **项目守护者**识别了三个可能打乱整个时间线的关键瓶颈角色
- **增长黑客**设计了针对空间计算固有可分享性的病毒式循环
- **支持响应者**将产品自身的AI能力转化为支持差异化因素

结果是一份全面的、跨职能的产品计划，可以作为实际开发的基础——由协同工作的AI智能体代理机构在单次会话中产出。
