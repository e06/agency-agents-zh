# 示例

本目录包含示例输出，展示了如何协调代理机构的各智能体协同完成实际任务。

## 为什么需要这些示例

agency-agents 仓库定义了数十个专业智能体，涵盖工程、设计、营销、产品、支持、空间计算和项目管理等领域。但仅有智能体定义并不能展示当你**一次性部署所有智能体**执行单一任务时会发生什么。

这些示例回答了这个问题：*"当整个代理机构协同工作时，实际是什么样的？"*

## 目录

### [nexus-spatial-discovery.md](./nexus-spatial-discovery.md)

**内容：** 一场完整的产品探索实践，8 个智能体并行工作，评估一个软件机会并产出统一计划。

**场景：** 网络研究发现了一个位于 AI 智能体编排与空间计算交叉领域的机会。随后整个代理机构被同时部署，产出以下内容：

- 市场验证和竞争分析
- 技术架构（8 服务系统设计，含完整 SQL 架构）
- 品牌策略和视觉识别
- 市场推广和增长计划
- 客户支持运营蓝图
- UX 研究计划（含用户画像和用户旅程图）
- 35 周项目执行计划，含 65 个冲刺工单
- 空间界面架构规范

**使用的智能体：**
| 智能体 | 角色 |
|-------|------|
| 产品趋势研究员 | 市场验证、竞争格局 |
| 后端架构师 | 系统架构、数据模型、API 设计 |
| 品牌守护者 | 定位、视觉识别、命名 |
| 增长黑客 | 市场推广策略、定价、发布计划 |
| 支持响应者 | 支持层级、入驻流程、社区 |
| UX 研究员 | 用户画像、用户旅程图、设计原则 |
| 项目引导者 | 阶段计划、冲刺、风险登记 |
| XR 界面架构师 | 空间 UI 规范 |

**核心要点：** 所有 8 个智能体并行运行，产出了相互关联、相互引用的连贯计划，无需协调开销。该输出展示了代理机构在单次会话中从"发现机会"到"提供完整蓝图"的能力。

## 添加新示例

如果你进行了一个有趣的多智能体实践，可以考虑添加到这里。好的示例应该展示：

- 多个智能体围绕共同目标协作
- 代理机构能力的广度
- 智能体定义的现实适用性
